Safety & Integrity

Constitutional Governance

ADAAD's governance model enforces safety at every layer. Hard-class invariants are unconditional — no mutation, no agent, and no LLM prompt can bypass them.

⊙
HUMAN-0 Gate
Non-delegable human authority gate. New organ proposals (AOEP-0), constitutional amendments, and GA sign-offs require explicit Dustin Reid sign-off. No bypass path exists by design.
HUMAN-0 is non-delegable by design
⛔
Hard-Class Invariants
9 unconditional blocking constraints that cannot be overridden by any agent, LLM call, or autonomous process. Ledger-first (STAKE-0) is the founding Hard-class invariant — all governance events committed to ledger before any downstream action.
STAKE-0: ledger commit precedes action
⛓
Append-Only Ledger
Governance and mutation ledgers are strictly append-only. No retroactive modification. SHA-256 hash chains link all events. Tamper detection is formally proven through the CEPD module.
SHA-256 chain · CEPD tamper-proof
⚖
Constitutional Jury
The CJS (Constitutional Jury System) governs mutation acceptance through multi-agent deliberation with quorum enforcement, dissent recording, and deterministic ledger verdicts.
Quorum required · Dissent recorded
⟳
CEL · Evolution Loop
Every mutation traverses the Constitutional Evolution Loop: proposal → shadow execution → jury → constitution check → ledger commit → production. No short-circuit path exists.
Every prompt grounded constitutionally
🔍
Identity Ledger Attestation
ILA provides cryptographic binding of agent identity to ledger entries. ARS staking means agents put reputation at risk for every mutation vote. Identity fraud triggers automatic stake burn events.
ILA · ARS stake-burn on fraud

Get Started

Quickstart

ADAAD runs on ADAADell — your local Linux node — and connects to the governed GitHub remote. Aponi dashboard at port 8080.

# Clone & onboard
git clone https://github.com/InnovativeAI-adaad/adaad
cd adaad
python3 onboard.py

# Run the server
./adaadell_run.sh
→ Aponi dashboard at :8080

# Run test suite
pytest --tb=short -q
6354 passed · 0 failed
# adaad.pro DNS / nginx
sudo apt install nginx certbot -y

# /etc/nginx/sites-available/adaad.pro
server_name adaad.pro www.adaad.pro;
root /home/dust/ADAAD/public;
location /api/ { proxy_pass :8080/api/; }

sudo certbot --nginx -d adaad.pro

# Subdomains
aponi.adaad.pro → :8080 (dashboard)
api.adaad.pro → :8080/api/
docs.adaad.pro → GitHub Pages
