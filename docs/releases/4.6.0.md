# ADAAD 4.6.0 Release Notes — Phase 21: Telemetry Ledger & Governed Observability

**Release date:** 2026-03-09
**Baseline:** v4.5.0 · PR-20-REL · 2,714+ tests passing
**Phase:** 21 — Telemetry Ledger & Governed Observability

---

## Summary

Phase 21 converts the intelligence routing pipeline's ephemeral in-memory telemetry
into a durable, sha256-chained, queryable audit record — following the same append-only
ledger pattern established by `mutation_ledger.py` and `reviewer_reputation_ledger.py`.

---

## Validated guarantees (in-repo)

- `FileTelemetrySink` appends `routed_intelligence_decision.v1` events to a JSONL file
  with sha256 hash chaining; `verify_chain()` is fail-closed on any hash mismatch.
- `TelemetryLedgerReader` provides deterministic query by strategy_id and outcome,
  `win_rate_by_strategy()`, `strategy_summary()`, and independent chain verification.
- `AutonomyLoop` accepts `telemetry_ledger_path` kwarg; `FileTelemetrySink` activates
  when a path is configured; `InMemoryTelemetrySink` remains the unchanged default.
- `ADAAD_TELEMETRY_LEDGER_PATH` env var respected at `AutonomyLoop` construction;
  explicit kwarg takes precedence.
- `GET /telemetry/decisions` endpoint: bearer-auth-gated (`audit:read`), read-only,
  supports `strategy_id` / `outcome` filters and `limit` / `offset` pagination;
  `sink_type` field indicates whether file or memory sink is active.
- 61 new tests; total test count: 2,775+.

## Explicit roadmap exclusions (not v4.6.0 guarantees)

- Telemetry aggregation dashboards (higher-order analytics on verified history).
- Cross-node telemetry federation.
- Telemetry-driven constitutional feedback (Phase 22 scope).
