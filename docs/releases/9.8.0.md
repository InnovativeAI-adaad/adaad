# ADAAD v9.8.0 — Phase 73: Seed Review Decision + Governance Wire

**Released:** 2026-03-14
**Phase:** 73 — Seed Review Decision + Governance Wire
**Requires:** v9.7.0 (Phase 72 complete)
**Seed Lifecycle Phase:** 3 of 5

---

## Summary

Phase 73 delivers the governed human-approval workflow for promoted seeds. `record_review()` enforces a non-empty `operator_id` gate before any status mutation (`SEED-REVIEW-HUMAN-0`), writes `SeedReviewDecisionEvent` to the lineage ledger before mutating state (`SEED-REVIEW-0`), carries a deterministic `decision_digest` (`SEED-REVIEW-AUDIT-0`), and is idempotent on terminal status (`SEED-REVIEW-IDEM-0`). Aponi gains live Approve/Reject buttons and real-time status reflection.

---

## What Shipped

### record_review()

| Capability | Invariant | Detail |
|---|---|---|
| Human operator gate | `SEED-REVIEW-HUMAN-0` | Non-empty `operator_id` required; empty raises `SeedReviewOperatorError` |
| Ledger-first write | `SEED-REVIEW-0` | `SeedReviewDecisionEvent` written to `LineageLedgerV2` before status mutation |
| Deterministic digest | `SEED-REVIEW-AUDIT-0` | SHA-256 `decision_digest` over (seed_id, operator_id, decision, timestamp) |
| Idempotency | `SEED-REVIEW-IDEM-0` | Second call on terminal status (approved/rejected) returns existing record |
| Bus emit | `SEED-REVIEW-BUS-0` | `seed_review_decision` frame emitted after ledger write |
| Elevated auth | `SEED-REVIEW-AUTH-0` | `require_audit_write_scope()` gates the review endpoint |

### New API Endpoint

`POST /innovations/seeds/promoted/{seed_id}/review` — accepts `operator_id`, `decision` (approved/rejected), optional `notes`
