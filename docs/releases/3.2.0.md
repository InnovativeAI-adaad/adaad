## Summary

ADAAD v3.2.0 ships Phase 7 — Reviewer Reputation & Adaptive Governance Calibration.
The system now closes the empirical feedback loop between human reviewer decisions and
constitutional calibration. Reviewer reputation — derived from latency, override rate,
mutation impact, and governance alignment — adjusts required reviewer count within
per-tier bounds while the constitutional floor (at least one human reviewer) is
architecturally inviolable.

This is the first release in which ADAAD's governance pressure adapts empirically to
reviewer track record. It is also the first release with a new constitutional rule
(`reviewer_calibration`, v0.3.0) since Phase 5 federation extensions shipped in v3.0.0.

## Shipped Milestones

### M7-01 — Reviewer Reputation Ledger (PR-7-01)

`runtime/governance/reviewer_reputation_ledger.py`

Append-only, SHA-256 hash-chained ledger recording every reviewer decision.
Decision taxonomy: `DECISION_APPROVE`, `DECISION_REJECT`, `DECISION_TIMEOUT`,
`DECISION_OVERRIDE`. HMAC-derived `reviewer_id` tokens enforce privacy (no
plaintext PII in the ledger). Write-once per decision; no retroactive modification.

Evidence: `tests/governance/test_reviewer_reputation_ledger.py` — 45 tests.

### M7-02 — Reputation Scoring Engine (PR-7-02)

`runtime/governance/reviewer_reputation.py`

Deterministic composite score `r ∈ [0.0, 1.0]` from four ledger-derived dimensions:

| Dimension | Weight |
|---|---|
| Latency | 0.20 |
| Override rate | 0.30 |
| Long-term mutation impact | 0.30 |
| Governance alignment | 0.20 |

Epoch weight snapshots journaled before each scoring pass; mid-epoch changes blocked.
`scoring_algorithm_version = "1.0"` bound in every event and epoch context.

Evidence: `tests/governance/test_reviewer_reputation.py` — 23 tests.
`tests/governance/test_pr_lifecycle_reviewer_outcome.py` — 14 tests.

### M7-03 — Tier Calibration Engine (PR-7-03)

`runtime/governance/review_pressure.py`

Adjusts reviewer count per tier based on aggregate cohort reputation:

- `composite_score >= 0.80` → count reduced by 1 (within tier min bound)
- `composite_score <= 0.40` → count increased by 1 (within tier max bound)

`CONSTITUTIONAL_FLOOR_MIN_REVIEWERS = 1` architecturally enforced. `validate_tier_config()`
raises `ValueError` at boot if any tier violates the floor — fail-closed.

### M7-04 — Constitution v0.3.0: `reviewer_calibration` Rule (PR-7-04)

`docs/CONSTITUTION.md` at **v0.3.0**. New advisory rule in `runtime/governance/constitution.yaml`:

```yaml
- name: reviewer_calibration
  severity: advisory
  validator: reviewer_calibration
```

Advisory: surfaces calibration telemetry in governance events; does not block mutations.
Escalatable to `blocking` via `ADAAD_SEVERITY_ESCALATIONS` env variable without code change.

### M7-05 — Aponi Reviewer Calibration Endpoint (PR-7-05)

`GET /governance/reviewer-calibration` served by `runtime/api/runtime_services.reviewer_calibration_service()`.

Read-only; auth-gated to `audit:read` scope. Returns cohort summary, average reputation,
tier pressure, constitutional floor status, epoch ID, constitution version, and scoring
algorithm version. Returns 422 if `epoch_id` is absent.

## Constitutional Invariants Added (Phase 7)

| Invariant | Enforcement |
|---|---|
| `CONSTITUTIONAL_FLOOR_MIN_REVIEWERS = 1` | `validate_tier_config()` fail-closed at boot |
| Ledger write-once | `append_entry()` raises if duplicate decision key exists |
| Mid-epoch weight lock | Scoring always binds to epoch-snapshotted weights |
| `scoring_algorithm_version` binding | Version recorded on every event; replay binds to it |
| No auto-approval path | Calibration adjusts count only; `GovernanceGate` sole approval surface |
