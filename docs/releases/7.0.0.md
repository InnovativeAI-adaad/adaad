## Summary

ADAAD v7.0.0 ships the Public API Consolidation (Phase 20) together with the
MarketSignalAdapter live bridge (Phase 46) and the complete operator endpoint surface.
This release closes the gap between the intelligence stack built in Phases 16–20 and
the production `server.py` API, wires live market fitness signals into the governance
health score, and brings all documentation into alignment with the v7.0.0 codebase.

This is the first release with a fully documented ROADMAP (42 phase sections covering
Phases 3–46), a complete amendment log in `docs/CONSTITUTION.md`, and zero residual
test failures from the 20-item target defect list.

## Shipped Milestones

### Phase 20 — Public API Consolidation

Gap analysis revealed nine modules and seventeen symbols from Phases 16–19 had never
been exported. All gaps closed:

- `AutonomyLoop` exported in `runtime.autonomy.__all__`
- `STRATEGY_TAXONOMY`, `CritiqueSignalBuffer`, `RoutedDecisionTelemetry`,
  `InMemoryTelemetrySink`, `EVENT_TYPE_ROUTED_INTELLIGENCE_DECISION` in
  `runtime.intelligence.__all__`
- `strategy.py.bak` deleted
- 18 import-contract tests in `tests/test_import_roots.py`

### Phase 46 — MarketSignalAdapter Live Bridge

`GET /evolution/market-fitness-bridge` returns live bridge status.
`test_market_fitness_bridge.py` — 20/20.
