# ADAAD 4.10.0 Release Notes — Phase 25: Pressure Adjustment Audit Ledger

**Release date:** 2026-03-09
**Baseline:** v4.9.0 · PR-24-REL · 2,915+ tests passing
**Phase:** 25 — Pressure Adjustment Audit Ledger

---

## Summary

Phase 25 makes `PressureAdjustment` permanent. Every advisory recommendation
is now immutably hash-chained — tamper-evident and queryable — completing the
governance advisory arc: emit → persist → verify → query.

---

## Validated guarantees (in-repo)

- `PressureAuditLedger`: append-only JSONL, SHA-256 hash chain; `PRESSURE_LEDGER_GENESIS_PREV_HASH = "sha256:" + "0"*64`; `emit()` isolates all I/O failures (WARNING only, never propagates); `chain_verify_on_open=True` raises `PressureAuditChainError` on any violation.
- `timestamp_iso` excluded from `record_hash` computation — same adjustment sequence → identical chain hashes regardless of wall-clock time (deterministic replay, test-verified).
- `PressureAuditReader`: `history()` newest-first, pressure_tier filter, limit≤500, offset; `tier_frequency()`; `tier_frequency_series(window)`; `verify_chain()`.
- `GET /governance/review-pressure`: emits to `PressureAuditLedger` when `ADAAD_PRESSURE_LEDGER_PATH` env is set; adds `ledger_active` and `ledger_sequence` to response; emit failure never propagates; inactive by default (no env → no file written).
- `GET /governance/pressure-history`: bearer-auth-gated, read-only; `pressure_tier` filter; `limit` 1–500; `422` on limit>500; `ledger_active: false` when no env or file.
- 50 new tests; total: 2,965+.
