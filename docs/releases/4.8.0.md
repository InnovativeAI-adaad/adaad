# ADAAD 4.8.0 Release Notes — Phase 23: Routing Health Signal Integration

**Release date:** 2026-03-09
**Baseline:** v4.7.0 · PR-22-REL · 2,822+ tests passing
**Phase:** 23 — Routing Health Signal Integration

---

## Summary

Phase 23 wires the Phase 22 `RoutingHealthReport` as a first-class signal
into `GovernanceHealthAggregator`, unifying the routing observability surface
with the composite governance health score.

---

## Validated guarantees (in-repo)

- `routing_health_score` is the fifth signal in `GovernanceHealthAggregator`
  (weight 0.15); the four original signals are rebalanced; total weight sum
  remains exactly 1.0 (validated by test).
- `_collect_routing_health()` defaults to `1.0` when no engine is wired
  (same convention as `federation_divergence_clean` single-node default);
  clamps output to `[0.0, 1.0]`; never propagates exceptions.
- `HealthSnapshot.routing_health_report` carries the serialized
  `RoutingHealthReport` dict when an engine is active; `None` otherwise.
- `GET /governance/routing-health` — bearer-auth-gated (`audit:read`), read-only;
  `window_size` 10–10000; `available: false` degraded mode when no file sink.
- `GET /governance/health` response gains `routing_health` field
  (available, status, health_score, dominant_strategy, report_digest,
  analytics_version); all existing fields and `schema_version` unchanged.
- `GovernanceGate` retains sole mutation-approval authority; routing health
  is advisory signal input only.
- 45 new tests; total: 2,867+.
