## Summary

v9.17.0 closes the gap opened in Phases 81–85: the entire evolution reasoning
stack is now wired into the live `ConstitutionalEvolutionLoop`. The Phase 64

## Track A — CEL Evolution Engine Wiring (PR-86-01)

### Step 8 — Real Fitness Scoring

`FitnessOrchestrator.score()` (5-component weighted composite) replaces the
`0.65` stub. `FitnessDecayScorer.evaluate()` applies temporal half-life
discount to scores. `CausalFitnessAttributor.attribute()` computes per-op
Shapley contributions. All three calls are ledger-recorded before
`fitness_summary` is written to state (`STEP8-LEDGER-FIRST-0`). All three
are fail-safe: any component failure falls back gracefully without blocking.

### Step 9 — PARETO-SELECT (new)

`ParetoCompetitionOrchestrator.run_epoch()` replaces the `score > 0.5`
scalar threshold that has been the promotion gate since Phase 64. Multiple
candidates may now be promoted per epoch (one per Pareto niche). Frontier
digest is ledger-recorded before Step 10 (`CEL-PARETO-0`).

### Post-Epoch — SELF-DISCOVERY hook

`ConstitutionalSelfDiscoveryLoop` wired as a non-blocking post-epoch hook
firing every 5 completed epochs. Failure never blocks epoch completion
(`CEL-SELF-DISC-NONBLOCK-0`). All candidates are advisory:
`SELF-DISC-HUMAN-0` requires governor sign-off before any candidate enters
`CONSTITUTION.md`.

### Constructor

`_UNSET` sentinel distinguishes lazy-import (default) from explicit `None`
injection, enabling full testability of all Phase 86 components in isolation.
`self._epoch_seq` tracks completed epochs; blocked epochs do not increment.

**New invariants:** `STEP8-LEDGER-FIRST-0`, `STEP8-DETERM-0`, `CEL-PARETO-0`,
`CEL-PARETO-DETERM-0`, `CEL-SELF-DISC-0`, `CEL-SELF-DISC-NONBLOCK-0`,
`SELF-DISC-HUMAN-0`

## Track B — CompoundEvolutionTracker (PR-86-02)

`runtime/evolution/compound_evolution.py` — `CompoundEvolutionTracker`:

The originally planned Phase 81 deliverable, now buildable with the full
Phase 79–83 infrastructure in place. `track_epoch()` accepts a
`ParetoCompetitionResult`, `MultiGenLineageGraph`, and per-candidate
`CausalAttributionReport` map, and produces a deterministic
`CompoundEvolutionRecord` aggregating fitness across the full ancestor chain
with generation-discounted weighting (`GENERATION_DISCOUNT_FACTOR = 0.8`).

`CompoundEvolutionRecord` carries: `record_id` (stable key), `compound_fitness`
(weighted aggregate ∈ [0,1]), `ancestry_trace` (root-first epoch chain),
`ancestor_contributions` (per-generation breakdown), `top_causal_ops`
(sorted Shapley contributors), `attribution_digests` (candidate → report_digest),
and `record_digest` (canonical determinism proof). Full `to_dict/from_dict`
round-trip preserves all fields.

**New invariants:** `COMP-TRACK-0`, `COMP-ANCESTRY-0`, `COMP-GOV-WRITE-0`,
`COMP-CAUSAL-0`
