## Track A — Constitutional Governance Infrastructure

### Abstract Interface Enforcement

| File | Change | Invariant |
|---|---|---|
| `runtime/evolution/event_signing.py` | `EventSigner` / `EventVerifier` → `ABC` + `@abstractmethod` | `EVENT-SIGN-ABSTRACT-0` |
| `runtime/innovations.py` | `GovernancePlugin.evaluate()` → `ABC` + `@abstractmethod` | `GPLUGIN-ABSTRACT-0` |
| `runtime/fitness_pipeline.py` | `FitnessEvaluator` abstractmethod contract clarified | — |

`NotImplementedError` stubs in production-path governance modules are a constitutional debt item — they allow a module to be instantiated in a partially-governed state with no compile-time or import-time error. Converting to proper `ABC` closes this gap: any attempt to instantiate without a concrete implementation raises `TypeError` at instantiation, not silently at runtime.

**New invariants:**

| Invariant | Requirement |
|---|---|
| `EVENT-SIGN-ABSTRACT-0` | `EventSigner` and `EventVerifier` are abstract; direct instantiation is a runtime error |
| `EVENT-SIGN-DETERM-0` | Implementations must be deterministic: identical (message, key) → identical `SignatureBundle` |
| `EVENT-SIGN-COMPARE-0` | Signature comparison must use `hmac.compare_digest` to prevent timing-oracle attacks |
| `GPLUGIN-ABSTRACT-0` | `GovernancePlugin` is abstract; direct instantiation is a runtime error |
| `GPLUGIN-DETERM-0` | `evaluate()` must be pure/deterministic: identical mutation inputs → identical result |
| `GPLUGIN-NOSIDE-0` | `evaluate()` must not mutate shared state or emit ledger events directly |

### Dual Webhook Handler Consolidation

`runtime/integrations/github_webhook_handler.py` previously contained a duplicate, ungoverned webhook implementation (115 lines) running in parallel with the governed `app.github_app` (318 lines). This was the C-03 remnant finding — two divergent code paths could process the same webhook event, with only one path enforcing `GITHUB-APP-GOV-0` and ledger recording.

**Resolution:** the legacy module is replaced with a governed shim that delegates 100% to `app.github_app`. All signature verification, event dispatch, GovernanceGate pre-checks, and ledger emission now flow through a single authoritative path.

| Invariant | Requirement |
|---|---|
| `WEBHOOK-SHIM-DELEG-0` | This module never re-implements signature verification or ledger emission |
| `WEBHOOK-SHIM-COMPAT-0` | Public API surface preserved for import compatibility |

**Closes:** `FINDING-AUDIT-C03` remnant (dual handler)
