# ADAAD v9.12.0 — Governance Hardening & Gate Recovery

**Released:** 2026-03-19
**Phase:** 77 remediation stream (post HUMAN-0)
**Requires:** v9.11.0

---

## Summary

v9.12.0 restores Tier 0/Tier 1 gate health after a live-audit block. This release hardens deterministic execution by removing dynamic-import patterns, re-establishes app-layer import boundaries through the `runtime.api` facade, and aligns release/governance evidence state for strict release validation.

---

## What Shipped

### Determinism Hardening

- Replaced forbidden inline `__import__()` execution in `runtime/capability_graph.py` with static top-level imports:
  - `import time as _time`
  - `import os as _os`
- Preserved existing behavior while satisfying determinism lint constraints.

### Import Boundary Recovery (`app/* -> runtime.api.*`)

- `app/main.py` now sources `validate_agent_contract_preflight` via `runtime.api.runtime_services`.
- `app/beast_mode_loop.py` now uses `runtime.api` facade exports for:
  - `SystemDeterminismProvider`
  - `emit_pr_lifecycle_event`
  - `validate_agent_contract_preflight`
- `app/github_app.py` now uses `runtime.api.app_layer` facade exports for:
  - `GovernanceGate`
  - external governance event recording
- `runtime/api/app_layer.py` and `runtime/api/runtime_services.py` were extended minimally to expose these facade-safe imports.

### Determinism/Recovery Test Corrections

- Fixed missing `monkeypatch` fixture injection in Ed25519 replay-attestation determinism tests (`tests/determinism/test_replay_attestation_determinism.py`) for:
  - happy path
  - tamper detection
  - unknown key id

### Lineage Integrity Invariant

- `runtime/evolution/lineage_v2.py` now invalidates `_verified_tail_hash` on append.
- Rationale: append mutates ledger state; trusted tail hash must be re-verified before reuse.
- Expected invariant: `get_verified_tail_hash()` is `None` immediately after append until `verify_integrity()` is re-run.

### Release & Evidence Alignment

- Added this release note: `docs/releases/9.12.0.md`
- Updated claims evidence matrix rows for Phases 58–61 from `Pending` -> `Complete`.
- Updated `.adaad_agent_state.json` version and registered live-audit failures as formal findings.

- Changes are conservative and scoped to audited failure surfaces.
