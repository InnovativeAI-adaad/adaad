# ADAAD v9.9.0 — Phase 74: Seed-to-Proposal Bridge

**Released:** 2026-03-14
**Phase:** 74 — Seed-to-Proposal Bridge
**Requires:** v9.8.0 (Phase 73 complete)
**Seed Lifecycle Phase:** 4 of 5

---

## Summary

Phase 74 bridges the Seed Lifecycle Pipeline to the `ProposalEngine`. `build_proposal_request()` converts an approved promotion queue entry into a `ProposalRequest` — enforcing an approved-only gate, writing a `SeedProposalEvent` to the lineage ledger before returning, and routing seed capability lanes to the correct `strategy_id`. Aponi gains a Propose button and live proposal toast.

---

## What Shipped

### build_proposal_request()

| Capability | Invariant | Detail |
|---|---|---|
| Approved-only gate | `SEED-PROP-0` | Non-approved seeds raise `SeedProposalNotApprovedError` |
| Human advisory | `SEED-PROP-HUMAN-0` | `ProposalRequest` carries `human_advisory` field — proposal must not be auto-submitted |
| Deterministic cycle_id | `SEED-PROP-DETERM-0` | SHA-256 over (seed_id, proposal_hash, timestamp_bucket) |
| Ledger-first write | `SEED-PROP-LEDGER-0` | `SeedProposalEvent` written to `LineageLedgerV2` before `build_proposal_request()` returns |
| Bus emit | `SEED-PROP-BUS-0` | `seed_proposal_generated` frame emitted after ledger write |
