## What is ADAAD?

It was conceived, designed, and built by **Dustin L. Reid** ŌĆö Founder of InnovativeAI LLC ŌĆö with one conviction: autonomous AI systems must be governed by hard rules, not soft suggestions.

## How the loop works

<p align="center">
  <img src="docs/assets/adaad-flow.svg" width="860" alt="ADAAD mutation pipeline: propose ŌåÆ compete ŌåÆ replay-verify ŌåÆ policy gate ŌåÆ execute ŌåÆ evidence"/>
</p>

Each epoch, three agents produce candidate mutations. Those candidates compete in a genetic-algorithm population ŌĆö crossed, mutated, and ranked by fitness. The fittest advance to the **GovernanceGate**: 18 rules, evaluated in order, fail-closed.

```
Propose  ŌåÆ  Compete (genetic selection)  ŌåÆ  Replay-verify  ŌåÆ  Constitutional gate  ŌåÆ  Execute  ŌåÆ  Evidence ledger
```

After each epoch, scoring weights self-calibrate via momentum gradient descent. Strategies that produce governable proposals gain weight. Underperformers decay. **The system learns which improvements are worth making ŌĆö inside the constitution, always.**

| Agent | Character | Focus |

| ­¤Éē **Beast** | Aggressive, performance-first | Throughput gains, complexity reduction, raw optimization |

All three agents compete through the same governed pipeline via UCB1 + Thompson Sampling bandit selection. Strategy arms that produce governable, high-fitness proposals accumulate selection pressure. Underperformers decay.

> [!IMPORTANT]

## Running an epoch

```python
from runtime.autonomy.ai_mutation_proposer import CodebaseContext
from runtime.evolution.evolution_loop import EvolutionLoop
import os

loop = EvolutionLoop(
    api_key=os.environ["ADAAD_CLAUDE_API_KEY"],
    generations=3,
)

context = CodebaseContext(
    file_summaries={"runtime/autonomy/mutation_scaffold.py": "Scoring engine."},
    recent_failures=[],
    current_epoch_id="epoch-001",
)

result = loop.run_epoch(context)

print(f"Proposals:   {result.total_candidates}")
print(f"Accepted:    {result.accepted_count}")
print(f"Next agent:  {result.recommended_next_agent}")
print(f"Accuracy:    {result.weight_accuracy:.1%}")
print(f"Duration:    {result.duration_seconds:.1f}s")
```

> [!NOTE]
> Every run is replay-verifiable. Use `--replay audit` to re-run any epoch and confirm byte-identical outputs. Divergence halts the pipeline and logs the exact delta.

### The 9 governance health signals

ADAAD tracks a composite health score `h Ōłł [0, 1]` over 9 signals that sum to exactly 1.00:

| Signal | Weight | Source Phase |
|:---|:---:|:---:|
| `avg_reviewer_reputation` | 0.18 | Ph. 7 |
| `amendment_gate_pass_rate` | 0.16 | Ph. 6 |
| `federation_divergence_clean` | 0.16 | Ph. 5 |
| `epoch_health_score` | 0.12 | core |
| `routing_health_score` | 0.10 | Ph. 23 |
| `admission_rate_score` | 0.09 | Ph. 26 |
| `governance_debt_health_score` | 0.08 | Ph. 32 |
| `certifier_rejection_rate_score` | 0.06 | Ph. 33 |
| `gate_approval_rate_score` | 0.05 | Ph. 35 |

`h < 0.60` triggers degraded-mode governance. `h < 0.40` issues an advisory epoch-pause signal. All signals are exposed via `GET /governance/health`. Weight sum is CI-verified at import time (`tolerance: 1e-9`).

### Determinism guarantees

- **Strict replay**: `ADAAD_REPLAY_MODE=strict` + `ADAAD_DETERMINISTIC_SEED=<seed>` ŌĆö every token, timestamp, and ID is deterministic. Divergence halts and is journaled. Enforced on both DreamMode and BeastModeLoop via `RuntimeDeterminismProvider` injection.
- **Determinism lock**: Set `ADAAD_DETERMINISTIC_LOCK=1` to freeze all non-deterministic paths and fail-fast on any seed-less operation at boot.
- **Hash-chained ledgers**: Every audit ledger uses SHA-256 hash-chaining. Tamper detection is automatic on open.
- **Fail-closed**: Missing `ADAAD_SOULBOUND_KEY` halts the pipeline immediately. No silent failures.
- **Weight bounds**: All scoring weights bounded to `[0.05, 0.70]`. No single signal can dominate.
- **Market bridge fail-closed**: `MarketSignalAdapter` errors are caught and logged ŌĆö `evaluate()` never raises. Synthetic baseline always available offline.

## Governance & Determinism Guarantees (Current State)

These are CI-verified on every merge. They cannot be bypassed:

- **Constitution v0.7.0** ŌĆö 18 rules active. `soulbound_privacy_invariant`, `bandit_arm_integrity_invariant`, and `market_signal_integrity_invariant` added in Phases 9, 11-A, and 13 respectively ŌĆö all BLOCKING.
- **Weight sum invariant** ŌĆö All 9 `SIGNAL_WEIGHTS` sum to exactly `1.00` (tolerance: `1e-9`). Verified at import time.
- **Weight bounds** ŌĆö Every weight is clamped to `[0.05, 0.70]`. No signal can dominate composite health.
