## What is ADAAD?

**ADAAD** (Autonomous Development & Adaptation Architecture) is an open-source AI governance and code mutation engine built by **[InnovativeAI LLC](https://github.com/InnovativeAI-adaad)**.

It was conceived, designed, and built entirely by **Dustin L. Reid** тАФ Founder of InnovativeAI LLC тАФ with one conviction: autonomous AI systems must be governed by hard rules, not soft suggestions.

> *"Autonomy without accountability isn't progress тАФ it's drift. ADAAD makes every decision traceable, every mutation defensible, and every failure visible."*
> тАФ **Dustin L. Reid**, Founder, InnovativeAI LLC

ADAAD is not a code formatter, a linter, or a suggestion tool. It is a **self-improving runtime** that operates under constitutional law.

## How the loop works

<p align="center">
  <img src="docs/assets/adaad-flow.svg" width="860" alt="ADAAD mutation pipeline: propose тЖТ compete тЖТ replay-verify тЖТ policy gate тЖТ execute тЖТ evidence"/>
</p>

Each epoch, three agents produce candidate mutations. Those candidates compete in a genetic-algorithm population тАФ crossed, mutated, and ranked by fitness. The fittest advance to the **GovernanceGate**: 18 rules, evaluated in order, fail-closed.

```
Propose  тЖТ  Compete (genetic selection)  тЖТ  Replay-verify  тЖТ  Constitutional gate  тЖТ  Execute  тЖТ  Evidence ledger
```

After each epoch, scoring weights self-calibrate via momentum gradient descent. Strategies that produce governable proposals gain weight. Underperformers decay. **The system learns which improvements are worth making тАФ inside the constitution, always.**

| Agent | Character | Focus |
|:---|:---|:---|
| ЁЯПЫя╕П **Architect** | Systematic, structural | Long-term coherence, dependency cleanup, interface contracts |
| ЁЯТн **Dream** | Creative, exploratory | Novel approaches, experimental rewrites, lateral improvements |
| ЁЯРЙ **Beast** | Aggressive, performance-first | Throughput gains, complexity reduction, raw optimization |

All three agents compete on equal terms through the same governed pipeline. `GovernanceGate` evaluates proposals from all three identically.

> [!IMPORTANT]
> `GovernanceGate` is the **only** surface with mutation-approval authority. Fitness scores influence which candidates advance тАФ but approval belongs exclusively to constitutional evaluation. This is enforced architecturally, not by convention.

## Running an epoch

```python
from runtime.autonomy.ai_mutation_proposer import CodebaseContext
from runtime.evolution.evolution_loop import EvolutionLoop
import os

loop = EvolutionLoop(
    api_key=os.environ["ADAAD_CLAUDE_API_KEY"],
    generations=3,
)

context = CodebaseContext(
    file_summaries={"runtime/autonomy/mutation_scaffold.py": "Scoring engine."},
    recent_failures=[],
    current_epoch_id="epoch-001",
)

result = loop.run_epoch(context)

print(f"Proposals:   {result.total_candidates}")
print(f"Accepted:    {result.accepted_count}")
print(f"Next agent:  {result.recommended_next_agent}")
print(f"Accuracy:    {result.weight_accuracy:.1%}")
print(f"Duration:    {result.duration_seconds:.1f}s")
```

> [!NOTE]
> Every run is replay-verifiable. Use `--replay audit` to re-run any epoch and confirm byte-identical outputs. Divergence from the recorded run halts the pipeline and logs the exact delta.

## ЁЯЫбя╕П Governance at a glance

ADAAD's governance is architecture, not configuration. The hierarchy is strict and cannot be bypassed:

```
Constitution  тЖТ  Architecture Contract  тЖТ  ArchitectAgent Spec  тЖТ  PR Procession
```

| Layer | File | What it governs |
|:---|:---|:---|
| **Constitution v0.3.0** | [`docs/CONSTITUTION.md`](docs/CONSTITUTION.md) | 18 rules (9 globally blocking, 4 tier-conditional blocking). No agent, operator, or PR can override these. |
| **Architecture Contract** | [`docs/ARCHITECTURE_CONTRACT.md`](docs/ARCHITECTURE_CONTRACT.md) | Interface and boundary invariants across all modules. |
| **ArchitectAgent Spec** | [`docs/governance/ARCHITECT_SPEC_v3.1.0.md`](docs/governance/ARCHITECT_SPEC_v3.1.0.md) | Phase 6 governance baseline and hardening lane. |
| **PR Procession** | [`docs/governance/ADAAD_PR_PROCESSION_2026-03.md`](docs/governance/ADAAD_PR_PROCESSION_2026-03.md) | Controls PR order, CI tier, and closure state. |

### The 9 governance health signals

ADAAD tracks a composite health score `h тИИ [0, 1]` over 9 signals that sum to exactly 1.00:

| Signal | Weight | Source |
|:---|:---:|:---|
| `avg_reviewer_reputation` | 0.18 | `ReviewerReputationLedger` |
| `amendment_gate_pass_rate` | 0.16 | `AmendmentPipeline` |
| `federation_divergence_clean` | 0.16 | `FederatedSignalBroker` |
| `epoch_health_score` | 0.12 | Core epoch evaluator |
| `routing_health_score` | 0.10 | `RoutingHealthAggregator` |
| `admission_rate_score` | 0.09 | `AdmissionRateTracker` |
| `governance_debt_health_score` | 0.08 | `GovernanceDebtLedger` |
| `certifier_rejection_rate_score` | 0.06 | `CertifierScanLedger` |
| `gate_approval_rate_score` | 0.05 | `GateDecisionLedger` |

`h < 0.60` triggers degraded-mode governance. All signals are exposed via `GET /governance/health`.

### Determinism guarantees

- **Strict replay**: Set `ADAAD_REPLAY_MODE=strict` + `ADAAD_DETERMINISTIC_SEED=<seed>` тАФ every token, timestamp, and ID is deterministic. Divergence halts and is journaled.
- **Hash-chained ledgers**: Every audit ledger (gate decisions, certifier scans, reviewer reputation, threat scans, admission audit, mutation, pressure) uses SHA-256 hash-chaining. Tamper detection is automatic.
- **Fail-closed**: Missing `ADAAD_SOULBOUND_KEY` halts the pipeline immediately with `soulbound_key_absent.v1` journaled. No silent failures.
- **Weight bounds**: All scoring weights bounded to `[0.05, 0.70]`. No single signal can dominate.

## Governance & Determinism тАФ Current Invariants

These are CI-verified on every merge. They cannot be bypassed:

- **Constitution v0.3.0** тАФ 18 rules active. `governance_health_floor`, `soulbound_privacy_invariant`, `bandit_arm_integrity_invariant`, and `market_signal_integrity_invariant` are all BLOCKING.
- **Weight sum invariant** тАФ All 9 `SIGNAL_WEIGHTS` sum to exactly `1.00` (tolerance: `1e-9`). Verified at import time.
- **Weight bounds** тАФ Every weight is clamped to `[0.05, 0.70]`. No signal can dominate composite health.
- **Fail-closed soulbound** тАФ `ADAAD_SOULBOUND_KEY` absent тЖТ immediate halt with journaled evidence. No default key, no bypass.
- **Replay determinism** тАФ `ADAAD_REPLAY_MODE=strict` requires `ADAAD_DETERMINISTIC_SEED`. All providers (clock, token, ID) are injected and seeded тАФ not pulled from system entropy.
- **`GovernanceGate` isolation** тАФ No module except the established gate chain may call `GovernanceGate` directly. Architecturally enforced.
- **Dispatch latency budget** тАФ Default `50.0 ms` (`ADAAD_DISPATCH_LATENCY_BUDGET_MS`). Excess triggers a governance audit event.
- **PolicyPromotionController** тАФ Auto-rollback on regression. No degraded weights are ever promoted.
