# Day-0 Procurement Fastlane Checklist (Shareable)

> Send this file in every new deal on day 0.  
> Audience: Buyer security, legal, procurement, and IT stakeholders.

## How Sales Should Use This

1. Copy this checklist into the deal room/email.
2. Attach the five linked files below.
3. Replace `[[CUSTOMER-SPECIFIC]]` placeholders before sending.
4. Route any legal redlines through Legal before sharing revised terms.

## Day-0 Package Contents

- [ ] `SECURITY_QUESTIONNAIRE_MASTER_ANSWERS.md`  
  Baseline answers for common security questionnaires.
- [ ] `DPA_MSA_FALLBACK_CLAUSES.md`  
  Fallback language blocks to accelerate legal redline cycles.
- [ ] `ARCHITECTURE_SECURITY_ONE_PAGER.md`  
  Buyer-facing architecture and security overview.
- [ ] `COMPLIANCE_EVIDENCE_INDEX.md`  
  Index of available evidence artifacts and owner/source.
- [ ] `SLA_SLO_SHEET.md`  
  Reliability targets, support SLAs, and service credit structure.

## Optional Add-Ons (Send Only If Requested)

- [ ] Current legal templates (MSA, DPA, security exhibit)
- [ ] Subprocessor list and privacy notice
- [ ] Penetration testing summary (NDA-gated)
- [ ] Additional architecture deep-dive materials

## Sales Send Template

Subject: `ADAAD Day-0 Procurement Packet — [[Customer Name]]`

Body:

- Attached is our day-0 procurement fastlane package to support your security/legal/procurement review.
- Primary package includes security questionnaire responses, legal fallback clauses, architecture/security overview, compliance evidence index, and SLA/SLO sheet.
- We can provide additional NDA-gated artifacts as needed.
- Please send your questionnaire and redline preferences, and we will route promptly.

## Internal Routing Checklist (Before External Send)

- [ ] Sales validated customer name and reviewer contacts.
- [ ] Security confirmed technical accuracy for any customer-specific claims.
- [ ] Legal approved any clause customizations.
- [ ] Latest file versions confirmed in `docs/commercial/procurement_fastlane/`.
